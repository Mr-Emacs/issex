ISSEX(1)

NAME

issex - a lightweight file-based issue tracker

SYNOPSIS

issex command [ flags ]

DESCRIPTION

issex manages issues stored as Markdown files on disk. Each issue lives in its own directory under a notes/ folder, which is located by walking up from the current working directory until one is found (similar to how git locates a repository root). Each issue directory contains a single notes.md file holding its metadata and free-form notes.

COMMANDS

add
Create a new issue.
-name \1
( required ) Short title for the issue.
-priority \1
Integer priority level. Defaults to R 0 .
-notes \1
Free-form notes to embed in the issue file.
list
List issues, optionally filtered by a query expression (see R QUERY\ SYNTAX below).
-src \1
Filter expression. Omit to list all issues.
close
Mark an issue as CLOSED.
-id \1
( required ) The HUID (human-readable unique ID) of the issue to close. HUIDs are printed by R list .

QUERY SYNTAX

Queries are small boolean expressions passed to R list\ -src . Whitespace is ignored.

Atoms

I p= n
Priority equals R n .
I p> n
Priority greater than R n .
I p< n
Priority less than R n .
s=1
Status is OPEN.
s=0
Status is CLOSED.
huid_or_title
Match a specific issue by its HUID or title.

Operators

B expr \ & \ expr
Logical AND (higher precedence than R | ).
B expr \ | \ expr
Logical OR.
I ! expr
Logical NOT. Applies to the immediately following atom or parenthesised group.
I ( expr )
Grouping.

EXAMPLES

Add an issue with priority 3:


issex add -name "Fix segfault in parser" -priority 3 -notes "Happens on empty input"

List all issues:


issex list

List all open issues with priority greater than 1:


issex list -src "p>1 & s=1"

List issues that are closed OR have priority 0:


issex list -src "s=0 | p=0"

List all open issues that are not priority 5:


issex list -src "s=1 & !p=5"

Close an issue by its HUID:


issex close -id task_20260417_143022

NOTES\ FILE\ FORMAT

Each issue is stored as a Markdown file with the following structure:


# <title>

- ID: <huid>

## PRIORITY: <n>

## STATUS: OPEN|CLOSED

## NOTES:

<free-form text>

The HUID is derived from the issue name and the creation timestamp ( name_YYYYMMDD_HHMMSS ).

ENVIRONMENT

No environment variables are used.

EXIT STATUS

0
Success.
1
Missing required argument or invalid usage.
-1
A command-level error occurred (e.g., issue not found, filesystem error, parse error).

BUILDING

issex ships with a self-rebuilding build script based on R nob.h . Compile the bootstrap once and it will keep itself up to date:


cc -o nob nob.c && ./nob

This produces the issex binary (or issex.exe on Windows). To enable Emacs-friendly output (file:line references), pass -DEMACS_PRINT at compile time (already the default in R nob.c ).
